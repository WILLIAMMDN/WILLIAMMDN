# GitHub Achievements Roadmap

GitHub achievements are earned from real GitHub events, not from local terminal history alone. This page tracks a clean path to earn them while keeping the profile honest and reviewable.

## What Counts

| Achievement | What usually unlocks it | Professional way to earn it |
| --- | --- | --- |
| Pull Shark | Merged pull requests | Build features in branches and merge PRs into your own repositories or collaborations. |
| Quickdraw | Closing an issue or PR very soon after opening it | Use issues for small tasks, then close them quickly when the fix is committed. |
| Pair Extraordinaire | Co-authored work in a merged PR | Pair with another developer or document real co-authored work in a PR. |
| Galaxy Brain | Accepted answers in GitHub Discussions | Help other users with useful answers in discussions. |
| Starstruck | A repository receives enough stars | Publish genuinely useful projects, docs or learning resources. |
| YOLO | Merging a PR without review | Can happen in personal repos, but it should not become the default workflow for serious projects. |
| Public Sponsor | Sponsoring open source publicly | Sponsor a maintainer or project through GitHub Sponsors. |

## What Does Not Count

- A local shell history by itself.
- A README section saying commands were used.
- Backdated commits unless they contain real work and are associated with your GitHub account.
- Empty commits created only to force visual activity.

## Suggested Plan

| Week | Action | Target |
| --- | --- | --- |
| 1 | Open small issues for README polish, docs, and project notes. | Issue discipline and Quickdraw opportunities. |
| 1-2 | Create feature branches and merge PRs into the profile repo and project repos. | Pull Shark progress. |
| 2-4 | Add real writeups for VialCentiva, Paltodoc, Academia Daemon and Cyber Labs. | Reviewable public work. |
| 4+ | Answer GitHub Discussions in areas you know: Git, React, Supabase, Linux, Python, AI basics. | Galaxy Brain progress. |
| Ongoing | Publish useful repos with clear READMEs, screenshots and setup steps. | Starstruck potential. |

## Notes

The historical archive documents previous work. Achievements should be earned through future GitHub events so the profile remains strong if someone reviews the underlying activity.
